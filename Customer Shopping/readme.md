🛒 Retail Customer Data Analysis using SQL
📌 Project Overview

This project focuses on analyzing retail customer transaction data using SQL to generate business insights related to revenue, customer behavior, subscription patterns, discounts, and product performance.

The analysis simulates real-world business intelligence queries typically required by stakeholders.

🛠 Tech Stack

PostgreSQL

SQL

Window Functions

CTEs

Aggregations

Subqueries

📊 Business Problems Solved
1️⃣ Revenue by Gender

Compared total revenue generated by male and female customers.

2️⃣ Discount vs Average Spend

Identified customers who used discounts but still spent above average.

3️⃣ Top 5 Highest Rated Products

Calculated average product ratings and ranked them.

4️⃣ Shipping Type Analysis

Compared average purchase amounts between Express and Free Shipping.

5️⃣ Subscriber vs Non-Subscriber Revenue

Measured:

Total customers

Average spend

Total revenue contribution

6️⃣ Products with Highest Discount Usage %

Calculated percentage of purchases made using discounts per product.

7️⃣ Customer Segmentation

Segmented customers into:

New

Returning

Loyal

Using CASE statements and CTEs.

8️⃣ Top 3 Products per Category

Used ROW_NUMBER() window function to rank products within categories.

9️⃣ Repeat Buyers vs Subscription Behavior

Analyzed if customers with more than 5 purchases tend to subscribe.

🔟 Revenue by Age Group

Measured revenue contribution by each age segment.

Performing customer segmentation using CASE logic

Translating business questions into SQL logic
